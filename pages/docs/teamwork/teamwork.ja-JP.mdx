# チームワーク

## 0. 序文

左上隅の「チームの作成または参加」をクリックして、チームインターフェイスに入ります。

!['teamwork'](/image/teamwork/teamwork01.png)

チームを作成するか、チームに参加するかを選択します。

!['teamwork'](/image/teamwork/teamwork02.png)

「チームの作成」: 組織名、地域、業種、役割を入力し、「作成」ボタンをクリックしてチームを作成します。

!['teamwork'](/image/teamwork/teamwork03.png)

「チームに参加」：名前、組織番号、応募理由（任意）を記入し、「応募」ボタンをクリックしてチームへの参加を申請してください。

!['teamwork'](/image/teamwork/teamwork04.png)

左上隅のアバターを再度クリックし、「パーソナル エディションに戻る」を選択して個人プラットフォームに戻ります。

!['teamwork'](/image/teamwork/teamwork05.png)

## 1. 概要
このドキュメントでは、データベース チーム管理インターフェイスの機能の使用方法について説明します。インターフェースは主に、**チーム設定**、**メンバー管理**、**承認リスト**、**権限管理**、**SQL監査**の4つの主要モジュールに分かれています。

!['teamwork1'](/image/teamwork/teamwork1.png)

## 2. チームマネジメント
### 2.1 チーム設定
チーム設定モジュールは主にチームの基本情報を管理および設定するために使用されます。ユーザーはこのモジュールで次の操作を実行できます。
・チーム情報編集：組織名情報を変更します。 **スーパー管理者**のみがチームのアバターと組織名を変更する権限を持っています。組織番号は一意であり、一度作成すると変更できません。
- 組織の譲渡: チームの所有権を他のメンバーに譲渡します。 **スーパー管理者のみ**が転送権限を持っています。

!['teamwork2'](/image/teamwork/teamwork2.png)

- サブスクリプション プラン: チーム PRO パッケージをアップグレードして、より多くのメンバーを招待できます。 **すべてのメンバー** がパッケージをアップグレードできます。最小サブスクリプションシート数は 5 名以上です。 （当初の3席からの5名増ではなく、合計5席となります）

初めて注文すると、システムのデフォルトのパッケージは現在の時刻から開始され、再度購入すると、最初に購入した時刻に基づいて更新されます。時間を延長するのと同じです。

> シートの更新と追加の違いに注意！ ！ ！

!['teamwork3'](/image/teamwork/teamwork3.png)

次回購入時はデフォルトで最新パッケージの最後から計算が開始されます。
シートを追加したいだけの場合は、「メンバー管理」インターフェースでシートをアップグレードしてください。

!['teamwork4'](/image/teamwork/teamwork4.png)

### 2.2 メンバー管理
メンバー管理モジュールは、チーム内のメンバーを管理するために使用されます。ユーザーは次の操作を実行できます。
- メンバーリストの表示: チーム内のすべてのメンバーの詳細を表示します。 **スーパー管理者**は、承認プロセス中であるがまだチームに参加していないメンバーを確認できます (表示は無効です)。

!['teamwork5'](/image/teamwork/teamwork5.png)

- メンバー情報の編集: **スーパー管理者**は、一般メンバーを管理者に設定したり、管理者資格を解除したりできます。

!['teamwork6'](/image/teamwork/teamwork6.png)

!['teamwork7'](/image/teamwork/teamwork7.png)

- メンバーを招待してシートをアップグレードする: **スーパー管理者と管理者**は、リンクをコピーして送信し、メンバーの数がいっぱいになった後にメンバーを招待できます。非会員の場合のチーム人数は3人です。非アクティブなメンバーは正式なメンバーの席を占めません。

!['teamwork8'](/image/teamwork/teamwork8.png)

初回パッケージ加入時のみ、一定期間の人数増加に相当する「アップグレード席」がございます。

> シートの更新と追加の違いに注意！ ！ ！

!['teamwork9'](/image/teamwork/teamwork9.png)

- ユーザーの削除: **スーパー管理者と管理者のみ**がチームからメンバーを削除できます。まだチームに参加していないメンバー、つまりまだ有効になっていないメンバーは削除できません。

!['teamwork10'](/image/teamwork/teamwork10.png)

### 2.3 購読記録
サブスクリプション レコード モジュールは、チーム メンバーの操作サブスクリプション レコードを表示するために使用されます。 **すべてのメンバー**は、ここですべてのメンバーのサブスクリプションステータスとそれに対応する操作記録を確認できます。

!['teamwork11'](/image/teamwork/teamwork11.png)

## 3. 承認リスト（一般ユーザーは閲覧できません）
承認リスト モジュールは、チームへの参加申請やデータ許可の申請など、チーム メンバーからのさまざまな申請を処理およびレビューするために使用されます。承認リストには次の列が含まれます。
- 承認フロー ID: 各承認リクエストの一意の識別子。
- 承認名: 承認リクエストの名前または説明。
- 承認ステータス: 現在の承認リクエストのステータス (例: 承認保留中、承認通過、承認拒否) が表示されます。
- アクション: 承認リクエストの詳細を表示します。

!['teamwork12'](/image/teamwork/teamwork12.png)

ユーザーはこのモジュールで次の操作を実行できます。
- 承認リクエストの表示: **スーパー管理者と管理者のみ**が、同意した承認リクエストを表示できます。簡単に言うと、申請リクエストに同意した人は、対応する承認レコードをインターフェイスに表示します。
- リクエストの承認または拒否: メンバーのリクエストを確認して処理します。 **スーパー管理者および管理者**のみが操作できます。

!['teamwork'](/image/teamwork/teamwork06.png)

!['teamwork'](/image/teamwork/teamwork07.png)

!['teamwork'](/image/teamwork/teamwork08.png)

## 4. 権限管理
権限管理モジュールは、チーム メンバーの権限を管理するために使用され、権限リストとアプリケーション リストの 2 つのサブモジュールに分かれています。
### 4.1 権限リスト(管理者)
権限リストは、チーム メンバーに付与された権限を表示および管理するために使用されます。ユーザーは次の操作を実行できます。
- 承認されたメンバーの表示: **スーパー管理者と管理者のみ**が、すべての承認されたメンバーとその詳細情報を表示できます。正規会員以外は閲覧できません。原理は「承認リスト」と同じです。

!['teamwork13'](/image/teamwork/teamwork13.png)

!['teamwork14'](/image/teamwork/teamwork14.png)


- 承認された管理者: **スーパー管理者と管理者のみ**が、承認が必要なユーザー、ユーザーがアクセスする必要があるデータ ソース、および権限が永続的であるかどうかを選択できます。 「送信」ボタンをクリックして、認証情報をシステムに保存します。


!['teamwork15'](/image/teamwork/teamwork15.png)

!['teamwork16'](/image/teamwork/teamwork16.png)

- 承認されたデータ権限: **スーパー管理者と管理者のみ**が、承認が必要なユーザー、ユーザーがアクセスする必要があるデータ ソース、および権限の有効期限を選択できます。 「送信」ボタンをクリックして、認証情報をシステムに保存します。後続のバージョンでは、権限がスコープ、行フィルタリング、その他の権限にさらに詳細に分割される予定ですので、ご期待ください。

!['teamwork17'](/image/teamwork/teamwork17.png)

!['teamwork18'](/image/teamwork/teamwork18.png)

### 4.2 申込一覧（一般会員）
リクエスト リストは、チーム メンバーによる権限リクエストを表示および処理するために使用されます。ユーザーはここで次の操作を実行できます。

!['teamwork19'](/image/teamwork/teamwork19.png)

- アプリケーション リクエストの表示: **スーパー管理者と管理者のみ**が、保留中のすべての権限リクエストを表示できます。
- アプリケーションの承認または拒否: **スーパー管理者と管理者のみ**が、メンバーの権限アプリケーションを確認および処理できます。
- データ権限の申請: **スーパー管理者を除くすべてのメンバー**は、データ権限、アクセスする必要があるデータ ソース、および権限の有効期限を申請できます。後続のバージョンでは、権限がスコープ、行フィルタリング、その他の権限にさらに詳細に分割される予定ですので、ご期待ください。

!['teamwork20'](/image/teamwork/teamwork20.png)

!['teamwork21'](/image/teamwork/teamwork21.png)

## 5. SQL監査(一般メンバーは閲覧できません)
SQL 監査モジュールは、チーム メンバーの SQL 操作記録をレビューするために使用されます。スーパー管理者と管理者は、すべてのメンバーが実行した SQL レコード、操作時間、操作ユーザーの詳細をここで表示できます。一般会員には閲覧権限がありません。
