# 连接 Postgresql

用 Chat2DB 快速连接 Postgresql 服务器

## 第一步：新建连接

!['PGsql2'](/image/connection/PGsql2.png)

点击左侧数据库按钮，选择新建连接，选择 Postgresql


## 第二步：填写连接信息

!['PGsql'](/image/connection/PGsql.png)

- `Name`：连接名称自定义可以方便记忆
- `Env`：选择连接环境，测试、生产等用于区分不同环境的连接
- `Storge`：选择连接存储类型，目前支持本地(LOCAL)、云端(CLOUD)
- `Host`：Postgresql 服务器地址，可以是 IP 或者域名
- `Port`：Postgresql 服务器端口，默认 5432
- `Authentication`：Postgresql 认证方式，用户名密码或者不需要认证
- `User`：Postgresql 用户名
- `Password`：Postgresql 密码
- `Database`：Postgresql 数据库名称，选填，不填会连接到默认数据库
- `URL`：Postgresql 连接 URL，选填，不填会根据上面的信息自动生成
- `Driver`：Postgresql 驱动，选填，不填会根据 URL 自动识别, 也可以手动选择，手动选择后会显示驱动配置项
- `SSH`：是否使用 SSH 连接，选填，不填默认不使用, 选择后会显示 SSH 配置项
- `Advanced Configuration`：高级配置，选填，不填默认不使用, 选择后会显示高级配置项

## 第三步：下载驱动

!['driver'](/image/connection/driver.png)

如果没有选择驱动，需要下载 Postgresql 驱动，点击下载按钮`Download Driver`下载会下载最新的 Postgresql 驱动，也可以手动上传自己的驱动`Upload driver`

## 第四步：配置 SSH 通道

!['ssh'](/image/connection/ssh.png)
- `USE SSH`：是否使用 SSH 通道
- `SSH Host`：SSH 服务器地址
- `SSH Port`：SSH 服务器端口 默认 22
- `Authentication`：SSH 认证方式，用户名密码或者私钥
- `User`：SSH 用户名
- `Password`：SSH 密码
- `Test the SSH connection`：测试 SSH 连接是否正常

## 第五步：测试连接

!['ssh'](/image/connection/test.png)

点击左下角的测试按钮，测试连接是否正常，如果连接正常会显示连接成功，否则会显示连接失败，可以根据错误信息调整连接信息。

## 第六步：保存连接
!['ssh'](/image/connection/test.png)

测试连接通过后，点击保存按钮，保存连接信息，连接信息会显示在左侧的数据库列表中，点击连接信息可以查看连接详情，也可以删除连接。
