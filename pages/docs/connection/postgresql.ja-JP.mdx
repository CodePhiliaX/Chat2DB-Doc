# Postgresqlに接続する

Chat2DB を使用して Postgresql サーバーにすばやく接続する

## ステップ 1: 新しい接続を作成する

!['PGsql2'](/image/connection/PGsql2.png)

左側のデータベース ボタンをクリックし、新しい接続 を選択して、Postgresql を選択します。


## ステップ 2: 接続情報を入力します。

!['PGsql'](/image/connection/PGsql.png)

- `Name`：接続名をカスタマイズすると覚えやすくなります
- `Env`：接続環境、テスト、本番などを選択して、異なる環境での接続を区別します
- `Storge`: 接続ストレージのタイプを選択します。現在はローカル (LOCAL) とクラウド (CLOUD) をサポートしています。
- `Host`: Postgresql サーバー アドレス。IP またはドメイン名にすることができます。
- `Port`: Postgresql サーバー ポート、デフォルトは 5432
- `Authentication`: Postgresql 認証方法、ユーザー名とパスワード、または認証は不要です
- `User`: Postgresql ユーザー名
- `Password`: Postgresql パスワード
- `Database`: Postgresql データベース名、オプション。入力しない場合は、デフォルトのデータベースに接続します。
- `URL`: Postgresql 接続 URL、オプション、入力しない場合は、上記の情報に基づいて自動的に生成されます
- `Driver`: Postgresql ドライバー (オプション)。入力しない場合は、URL に基づいて自動的に識別されます。手動で選択することもできます。手動で選択すると、ドライバー設定項目が表示されます。
- `SSH`: SSH 接続を使用するかどうか、オプション。未入力の場合、デフォルトで無効になります。選択すると、SSH 設定項目が表示されます。
- `Advanced Configuration`: 詳細設定、オプション、未入力の場合はデフォルトでは使用されません。選択後に詳細設定項目が表示されます。

## ステップ 3: ドライバーをダウンロードする

!['driver'](/image/connection/driver.png)

ドライバーを選択しない場合は、Postgresql ドライバーをダウンロードする必要があります。ダウンロード ボタン`Download Driver`をクリックして、最新の Postgresql ドライバーをダウンロードします。また、独自のドライバー`Upload driver`を手動でアップロードすることもできます。

## ステップ 4: SSH トンネルを構成する

!['ssh'](/image/connection/ssh.png)
- `USE SSH`：SSHトンネルを使用するかどうか
- `SSH Host`：SSHサーバーアドレス
- `SSH Port`：SSHサーバーポートのデフォルトは22
- `Authentication`：SSH 認証方法、ユーザー名、パスワード、または秘密キー
- `User`：SSHユーザー名
- `Password`：SSHパスワード
- `Test the SSH connection`：SSH接続が正常かどうかをテストする

## ステップ 5: 接続をテストする

!['ssh'](/image/connection/test.png)

左下隅のテスト ボタンをクリックして、接続が正常であるかどうかをテストします。接続が正常であれば、接続は成功します。そうでない場合は、接続は失敗します。エラー メッセージに従って、接続情報を調整できます。

## ステップ 6: 接続を保存する
!['ssh'](/image/connection/test.png)

テスト接続が成功したら、保存 ボタンをクリックして接続情報を保存します。接続情報が左側のデータベース リストに表示されます。接続情報をクリックして接続の詳細を表示するか、接続を削除します。
