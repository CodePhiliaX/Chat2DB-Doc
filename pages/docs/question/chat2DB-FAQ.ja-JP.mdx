### Q: 科学的にインターネットをサーフィンしようとしているのに、openai を使用できないのはなぜですか?
A: Web gpt が正常にチャットできる場合は、VPN がグローバルではなくブラウザーにのみ有効であり、一部の Java アプリケーションには有効ではないことを意味します。chat2db クライアントでプロキシを手動で構成できます。

 > 設定が必要です:
 > * HTTP プロキシ ホストは通常 127.0.0.1 です。
 > * HTTP プロキシ ポート VPN ソフトウェア プロキシのローカル ポート番号は、Clash では 7890 です
 > * API ホスト 直接使用する openai サービスが仲介エージェントとして機能しない場合は、入力する必要はありません。



### Q: フロントエンド ページに白い画面が表示されます
A: 一部のフロントエンドのローカル ストレージ値は古いバージョンと新しいバージョンの間で互換性がないため、コンソールを開いてローカル ストレージをクリアしてみてください。後続のバージョンが v2.0.2 と互換性がない場合は、フロントエンドが自動的にクリアされます。ローカルストレージの内容。

### Q: フロントエンドは依存関係をインストールするために糸を実行しますが、Electron はスタックして応答しません。
A: 1. 画像を変更してみてください。 2. .npmrc の Electron 画像アドレスを変更してください ([参考記事](https://www.cnblogs.com/makalochen/p/16154510.html#%E8%A7%A3%E5%86%B3%E5%8A%9E%E6%B3%95))

### Q: 新しいリンクを作成した後、テーブルが空になります
A: リンクの上部にある対応するデータベースに切り替える必要があります。ここを参照してください -> https://github.com/codePhiliaX/Chat2DB/issues/439

### Q: データの視覚的な変更でエラーが報告されるのはなぜですか: 各 SOL ステートメントは 1 つのレコードのみを更新する必要があります。更新には unlque キーを使用してください。
A: テーブルに一意のキーがなく、まったく同じ長さのデータが N 個あり、そのデータの 1 つを変更した場合、どの部分を変更する予定かを知る方法は技術的にありません。
### Q: AI 関数エラー: 例外が発生しました。ヘルプの詳細ログを確認してください: CHAT2DB_SERVICE_ERROR
A: テーブルのフィールドが多すぎます。選択するテーブルをいくつか減らしてみてください。
