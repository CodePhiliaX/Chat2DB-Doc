# クイックスタート

## 📖 紹介する

Chat2DB は、AI ファーストのデータ管理、開発、分析ツールです。AIGC (人工知能生成コード) 機能を核としており、自然言語から SQL への変換、SQL から自然言語への変換、レポートの自動生成も可能で、担当者の効率を大幅に向上させます。 。 1つの製品でデータ管理、データ開発、データ分析の機能を実現でき、SQLを理解していなくてもビジネスデータのクエリやレポートの生成を迅速に行うことができます。

## 🚀 クイック起動

当社の製品は、Web ページとクライアントの 2 つの使用方法を提供します。 Web ページはすぐに使用でき、クライアントはよりスムーズなユーザー エクスペリエンスを提供します。同時に、一部のプライベート データベースやローカル データにパブリック ネットワークからアクセスできない場合、クライアントはデータベースにローカルに接続して、プロキシを必要とせずに直接送信できるため、シンプルで便利であり、シナリオに応じて Web ページの使用とクライアントの使用を柔軟に切り替えることができます。
以下に製品の使用方法を説明します。

WEBアクセスアドレス: [https://app.chat2db.ai](https://app.chat2db.ai), クライアントダウンロードアドレス: [https://chat2db.ai/download](https://chat2db.ai/download)


## ステップ 1: クライアントのダウンロード

Web バージョンを使用する場合は、この手順を無視して、[アプリケーション アドレス](https://app.chat2db.ai) に直接アクセスできます。

クライアント ダウンロード アドレス: [クライアント ダウンロード アドレス](https://chat2db.ai/download) 対応するシステムを選択してダウンロードしてインストールするだけです Windows、Mac、Linux をサポートしています。

## ステップ 2: 登録とログイン

電子メールから直接確認コードを取得して直接ログインすることも、サードパーティを使用してログインすることもできます。 現在、Github や Google などの複数のログイン方法をサポートしています。

[ログインアドレス](https://app.chat2db.ai/login)

Chat2DB は現在、次の 5 つのログイン方法をサポートしています。
1.Googleログイン
2.GitHub へのログイン
3.WeChatログイン

5. メールアドレスとパスワードを使用してログインします

## ステップ 3: データベース接続を作成する

データベース接続は Chat2DB を使用するための最初のステップです。後続の操作を実行するにはデータベースに接続する必要があります。 左側のデータベース接続をクリックし、左下隅の 新しい接続 をクリックして、使用するデータベースを選択し、データベース接続情報を入力して、保存 をクリックします。 現在サポートしておりますMysql、Postgresql、SQLServer、DB2、 Oracle、SQLServer、Oceanbase、Clickhouse、Redis、H2、SQLite、DM、KingBase および他のデータベース。
### 1. 新しい接続を作成します
!['newConnection-connection'](/image/connection/newConnection.png)

左側の データベース ボタンをクリックし、新しい接続 を選択し、必要なデータベースの種類を選択し、データベース接続情報を入力して、保存 をクリックします。

### 2. データベース接続情報を入力します。

!['newConnection-connection'](/image/connection/connectioninfo.png)

- `Name`：接続名をカスタマイズすると覚えやすくなります
- `Env`：接続環境、テスト、本番などを選択して、異なる環境での接続を区別します
- `Host`: Mysql サーバー アドレス。IP またはドメイン名にすることができます。
- `Port`: Mysql サーバー ポート、デフォルトは 3306
- `Authentication`: Mysql 認証方法、ユーザー名とパスワード、または認証は不要です
- `User`: MySQL ユーザー名
- `Password`: MySQL パスワード
- `Database`: Mysql データベース名、オプション。入力しない場合は、デフォルトのデータベースに接続します。
- `URL`: Mysql 接続 URL、オプション、入力しない場合は、上記の情報に基づいて自動的に生成されます
- `Driver`: Mysql ドライバー (オプション)。入力しない場合は、URL に基づいて自動的に識別されます。手動で選択することもできます。手動で選択すると、ドライバー設定項目が表示されます。
- `SSH`: SSH 接続を使用するかどうか、オプション。未入力の場合、デフォルトで無効になります。選択すると、SSH 設定項目が表示されます。
- `Advanced Configuration`: 詳細設定、オプション、未入力の場合はデフォルトでは使用されません。選択後に詳細設定項目が表示されます。

クライアント上のデータベースごとに異なるドライバーをダウンロードする必要があり、SSH 接続が必要な場合は、SSH 情報を入力する必要があります。


## ステップ 4: 接続をテストする

!['ssh'](/image/connection/test.png)

左下隅のテスト ボタンをクリックして、接続が正常であるかどうかをテストします。接続が正常であれば、接続は成功します。そうでない場合は、接続は失敗します。エラー メッセージに従って、接続情報を調整できます。 最後に保存をクリックします。

## ステップ 5: Chat2DB データベース開発を使用する

接続が成功したら、接続をダブルクリックするか、右クリックして接続を開始します。その後、Chat2DB でデータベースの追加、削除、変更、クエリを実行できます。また、AIGC 機能を使用してデータベースに直接クエリを実行することもできます。自然言語を使用するか、SQL を使用して自然言語に変換します。

### 1. データベース管理
新しく作成した接続をダブルクリックしてデータベース管理インターフェイスに入り、データベースの追加、削除、変更、クエリを実行できます。
!['databasemanage'](/image/connection-manage/databasemanage.png)

### 2.データテーブル管理
#### 新しい編集テーブルを作成する
!['tables'](/image/connection-manage/create_t_base.png)
新しいフォームを作成し、基本情報を入力します
- `Table Name`: テーブル名
- `Comment`: テーブルの注釈
- `Character set`: キャラクターセット
- `Engine`: ストレージエンジン
- `Increment value`: 自己増加開始値

!['tables'](/image/connection-manage/create_t_column.png)
列情報を入力します
- `Column Name`: リスト名
- `Type`: タイプ
- `Size`: 長さ
- `Nullable`: 空かどうか
- `Key`: 主キー
- `Comment`: コメント

!['tables'](/image/connection-manage/create_t_index.png)
- `Index Name`: インデックス名
- `Column`: リスト
- `Type`: インデックスの種類
- `Comment`: コメント

### 3. データクエリ
テーブル、テーブル (特定のテーブル) を右クリックして「クエリ コンソール」を選択し、SQL を通じて希望の条件のデータをクエリします。
!['open_table'](/image/connection-manage/new_console.png)


## ステップ 6: Chat2DB データ分析を使用する

!['resultDashboard'](/image/connection-manage/resultDashboard.png)

左側の`Dashboard`をクリックし、「+」記号をクリックして新しいレポートを作成し、レポート情報を入力して、必要なレポートの種類を選択して、保存をクリックします。

### 1. 新しいグラフを作成する

!['Dashboard'](/image/connection-manage/creatDashboard.png)

左側の`Dashboard`をクリックし、「+」記号をクリックして新しいレポートを作成し、レポート名 (必須) とレポートの説明 (オプション) を入力します。


### 2. レポート情報を入力します

!['Dashboard2'](/image/connection-manage/dashboard2.png)

`Create Chart`をクリックし、ページ上の`Data allcation`をクリックして、生成されたレポートに対応するデータベースを選択します。

!['Dashboard3'](/image/connection-manage/dashboard3.png)

選択したら、対応する SQL ステートメントを記述し、`Run`をクリックして、`Back to Chart`をクリックします。

!['Dashboard4'](/image/connection-manage/dashboard4.png)

例: Employees テーブルの情報をクエリしてレポートを作成したいと考えています。


### 3.レポートの種類を選択します

!['Dashboard5'](/image/connection-manage/dashboard5.png)

ここでレポート タイプを選択し、選択したタイプに応じて対応する変数 (水平軸と垂直軸の座標、パーセンテージなど) を決定します。


### 4. レポートを保存します。

!['Dashboard6'](/image/connection-manage/dashboard6.png)

選択したら`save`をクリックして保存してください。
